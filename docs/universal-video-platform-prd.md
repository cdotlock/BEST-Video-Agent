# 通用视频平台 PRD（V1）

## 1. 背景与目标

当前系统在“小说转视频”场景可用，但在风格控制、初始化流程、UI 易用性、能力裁剪和长期记忆方面偏向单一画风/单一流程。
本 PRD 目标是将平台升级为**可适配任意风格与多工作流的通用视频平台**，并保留“领域专项模块”作为可插拔能力层。

### 1.1 业务目标

- 支持用户在生成前完成风格确认（可搜索参考图 + 反推提示词）。
- 将生成链路从“固定模板”升级为“目标风格驱动 + 用户偏好驱动”。
- 将工作台升级为更简洁、高效、低学习成本的白色主题 UX。
- 精简低频能力，降低认知负担，提升首日可用性。
- 建立用户偏好与效果反馈闭环，实现“越用越懂用户”。

### 1.2 非目标

- 不做独立新系统；继续复用现有 service layer、MCP、skills、chat、task 基础设施。
- 不引入非 MCP 标准的私有工具协议。
- 不引入绕开 Zod 校验的快速通道。

## 2. 用户与核心场景

### 2.1 目标用户

- 需要快速产出视频内容的创作者/运营。
- 对画风一致性有要求，但不希望手写复杂提示词的用户。
- 希望通过历史项目沉淀个人“风格资产”的高频用户。

### 2.2 核心任务流（Happy Path）

1. 用户输入创作目标（主题/脚本/素材）。
2. 系统进入“风格初始化”向导：
   - 搜索参考图（可多轮）；
   - 选择参考图并保存到 domain resources；
   - 自动反推 style prompt（结构化字段）；
   - 用户确认/微调后锁定本次风格。
3. 进入任务执行：agent 按锁定风格 + 用户偏好执行 storyboard 与素材生产。
4. 结果交付后收集反馈（like/dislike + 文字点评），写入偏好记忆。
5. 下次任务自动注入偏好摘要，用户可覆盖。

## 3. 产品方案

## 3.1 方案 A：通用能力层 + 专项模块层

### A. 通用能力层（所有领域共用）

- 会话与任务执行：Task + SSE + 会话恢复。
- 资产仓库：domain_resources（按 category 自由分类，按 media_type 渲染）。
- 风格初始化服务：参考图搜索、选择、提示词反推、确认。
- 偏好记忆服务：长期偏好（style / pacing / framing / color）。

### B. 专项模块层（按领域裁剪）

- 继续保留 `video` 作为首个专项模块。
- 专项模块只提供：预设技能包、上下文注入器、领域 UI 组件。
- 严禁在专项模块重复实现通用 CRUD（必须走 service layer）。

## 3.2 风格初始化（新前置流程）

### 3.2.1 功能拆分

1. **Reference Search**：按关键词检索候选图。
2. **Reference Select**：用户确认参考图并入库。
3. **Prompt Reverse**：基于参考图反推可编辑提示词。
4. **Style Confirm**：用户确认后写入任务上下文。

### 3.2.2 数据契约（逻辑层）

- `style_profile`
  - `style_goal`: string
  - `reference_images`: string[]
  - `reverse_prompt`: string
  - `negative_prompt`: string
  - `constraints`: string[]
  - `version`: number

### 3.2.3 验收标准

- 若未完成 style confirm，不允许进入最终生成。
- 允许用户在任务中途回到 style confirm 并触发增量重生成。
- style_profile 持久化并可在后续会话复用。

## 3.3 Prompt 与编排升级

- 系统 prompt 从“固定画风说明”改为“风格槽位模板 + 偏好注入模板”。
- 增加“冲突处理规则”：
  1. 用户本轮显式要求 > 历史偏好；
  2. 风格锁定参数 > 默认模板；
  3. 安全与合规约束 > 所有创作偏好。
- Tool-use loop 在每轮任务开始时读取 style_profile + 偏好摘要。

## 3.4 UI 2.0（白色主题）

### 3.4.1 视觉原则

- 主色：白 + 中性灰；强调色用于状态/CTA。
- 版面：高信息密度但低视觉噪音。
- 组件：统一圆角、间距、层级与可点击反馈。

### 3.4.2 布局

- 左：最终交付物（storyboard / output timeline）
- 中：对话与任务状态（chat + streaming event）
- 右：素材资源（按 category 分组，可折叠）

### 3.4.3 交互增强

- 生成前向导（Step 1-4）固定在中栏顶部。
- 二级菜单：风格库、参考图、提示词版本、历史偏好。
- 常用操作一键可达：复用上次风格、锁定镜头语言、回滚版本。

## 3.5 能力精简（Skills/MCP）

- 只保留高频、可解释、可验证的内置能力。
- 下线原则：
  - 最近周期低调用；
  - 与核心任务无直接价值；
  - 可由通用能力替代。
- 下线需提供迁移文档：替代路径、影响范围、回滚方案。

## 3.6 记忆与自进化

### 3.6.1 记忆分层

- 短期记忆：当前 session 上下文与 style_profile。
- 中期记忆：最近 N 次任务偏好摘要。
- 长期记忆：稳定偏好画像（主题、画风、节奏、镜头、禁忌项）。

### 3.6.2 进化闭环

1. 任务完成后收集显式反馈（评分 + 标签 + 文本）。
2. 反馈入库并归并为偏好更新建议。
3. 下次任务启动时注入“偏好摘要 + 置信度 + 最近更新来源”。
4. 用户可一键重置或局部关闭偏好注入。

## 4. 约束与架构对齐

- API route / MCP provider 仅做协议转换与 Zod 校验，业务逻辑下沉到 `src/lib/services/`。
- 输入校验统一使用 Zod，禁止绕过 parse。
- 通用能力与专项模块保持“组合关系”，避免双轨实现。
- 新增/变更 API 时同步更新 `docs/api-playbook.md` 与对应 use case。

## 5. 里程碑（对应 ROADMAP）

- M1：信息架构重构与边界确认。
- M2：风格初始化链路可用（搜索、入库、反推、确认）。
- M3：通用 prompt 与偏好注入生效。
- M4：UI 2.0 首版上线。
- M5：Skills/MCP 精简与迁移说明完成。
- M6：记忆与反馈闭环上线。
- M7：文档与验证手册补齐，形成稳定回归基线。

## 6. 风险与缓解

- 风格搜索质量波动：增加“候选置信度 + 人工确认”环节。
- 记忆误导生成：所有偏好注入必须可见、可覆盖、可关闭。
- UI 改版扰动老用户：提供旧版入口灰度期与迁移指引。
- 能力精简误伤：按调用数据分批下线，保留快速回滚。
